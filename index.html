¿Que es PDISC?

Es el espacio de síntesis final donde los estudiantes aplican los conocimientos adquiridos (hardware, software, redes) para desarrollar una solución informática real.

¿Que temas se tratan?

  • Gestion y Administracion del Proyecto
  • Analisis y Levantamiento de Requerimientos
  • Diseño del Sistema
  • Implementacion y Codificacion
  • Pruebas y Control de Calidad
  • Despliegue, Entrega y Mantenimiento
  • Producto Final y Evaluacion

Gestión y Administración del Proyecto

La gestión de proyectos de software es la disciplina que organiza y administra los recursos para completar un proyecto de software dentro del alcance, tiempo y costo definidos. Implica la planificación, programación, asignación de recursos, ejecución, seguimiento y entrega del proyecto. A diferencia de la gestión tradicional, los proyectos de software suelen utilizar metodologías ágiles como Scrum o Kanban, que permiten adaptarse a los cambios y al feedback continuo del cliente. El gestor del proyecto es responsable de liderar al equipo (desarrolladores, testers, diseñadores), gestionar el presupuesto, mitigar riesgos y asegurar que se cumplan los hitos establecidos, Para Saber Mas.

Analisis y Levantamiento de Requerimientos

El levantamiento de requerimientos es el proceso de identificar y documentar las necesidades y expectativas de los usuarios y stakeholders (interesados) sobre el sistema que se va a construir. Esta etapa es crítica porque define qué debe hacer el software, previniendo errores costosos en fases posteriores. Existen dos tipos principales de requerimientos: los funcionales (acciones específicas que debe realizar el sistema) y los no funcionales (cualidades como rendimiento, seguridad o usabilidad). Las técnicas más comunes para obtener requerimientos incluyen entrevistas, encuestas, observación, análisis de documentación, talleres (workshops), tormenta de ideas e historias de usuario, Para Saber Mas.

Diseño del Sistema

En la fase de diseño se define la arquitectura y los componentes del software basándose en los requerimientos analizados. Esto incluye decisiones sobre la estructura del sistema (cliente-servidor, capas, microservicios), el diseño de la base de datos (modelo entidad-relación, normalización), y la creación de prototipos de interfaz de usuario (UI/UX). Una herramienta fundamental en esta etapa es el Lenguaje Unificado de Modelado (UML), que permite visualizar el sistema a través de diagramas como el de clases, secuencia, actividades o casos de uso, facilitando la comunicación entre desarrolladores y stakeholders, Para Saber Mas.

Implementacion y Codificacion

La implementación es la fase donde se escribe el código fuente del sistema según las especificaciones de diseño. Aquí se seleccionan las tecnologías (lenguajes de programación, frameworks, bases de datos) y se aplican buenas prácticas de codificación como los principios SOLID, estándares de nomenclatura y documentación. El equipo de desarrollo trabaja de manera colaborativa utilizando sistemas de control de versiones (como Git) que permiten gestionar cambios, crear ramas de desarrollo y realizar revisiones de código (code reviews) antes de integrar las funcionalidades al proyecto principal, Para Saber Mas.

Pruebas y Control de Calidad

Las pruebas de software tienen como objetivo verificar que el sistema funciona correctamente, cumple con los requerimientos y está libre de errores. Esta fase se realiza en paralelo al desarrollo e incluye diferentes niveles: pruebas unitarias (sobre componentes individuales), pruebas de integración (entre módulos), pruebas de sistema (completas) y pruebas de aceptación por el usuario (UAT). También se realizan pruebas no funcionales como las de rendimiento, carga, seguridad y usabilidad. El equipo de calidad documenta los defectos encontrados y los reporta a los desarrolladores para su corrección, repitiendo el ciclo hasta que el software esté listo, Para Saber Mas.

Despliegue, Entrega y Mantenimiento

El despliegue es el proceso de poner el software en el entorno de producción para que los usuarios finales puedan utilizarlo. Esto implica preparar servidores, configurar bases de datos, empaquetar la aplicación (por ejemplo, usando contenedores Docker) y, en muchos casos, automatizar el proceso mediante pipelines de Integración Continua/Despliegue Continuo (CI/CD). Una vez en producción, comienza la fase de mantenimiento, que incluye: correctivo (corrección de errores detectados), perfectivo (mejoras y adaptaciones a nuevos requisitos) y evolutivo (actualizaciones tecnológicas). El mantenimiento asegura que el software siga siendo útil y eficiente a lo largo del tiempo, Para Saber Mas.

Producto Final y Evaluacion

El producto final representa la culminación del proyecto de software, donde se entrega el sistema completamente funcional, documentado y listo para su uso por parte del cliente o los usuarios finales. Esta fase no se limita únicamente a la entrega técnica, sino que incluye una evaluación integral que verifica el cumplimiento de todos los objetivos planteados al inicio del proyecto, Para Saber Mas.
